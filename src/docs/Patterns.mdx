# 패턴 가이드

컴포넌트를 조합하여 실제 서비스 화면을 구성하는 방법입니다.

## 기본 패턴

### 입력 폼

사용자 데이터를 수집하는 가장 기본적인 패턴입니다.

```tsx
import { TextInput, Textarea, Select, Checkbox, Button } from "krds-community-react";

function InputForm() {
  return (
    <form className="fieldset">
      <TextInput label="제목" placeholder="제목을 입력해 주세요" />
      <Textarea label="내용" placeholder="내용을 입력해 주세요" maxCount={500} />
      <Select
        label="분류"
        options={[
          { value: "inquiry", label: "문의" },
          { value: "suggest", label: "건의" },
          { value: "complain", label: "불편신고" },
        ]}
      />
      <Checkbox label="개인정보 수집·이용에 동의합니다" />
      <div className="btn-wrap">
        <Button variant="tertiary">취소</Button>
        <Button variant="primary">제출</Button>
      </div>
    </form>
  );
}
```

**규칙:**
- 라벨은 인풋 위에 배치
- 필수 필드는 라벨에 `*` 표시
- 에러 메시지는 해당 필드 바로 아래
- 제출 버튼은 폼 하단 우측에 Primary

### 목록 탐색

### 확인 (되돌릴 수 없는 액션)

사용자에게 중요한 결정을 확인받는 패턴입니다.

```tsx
import { Modal, ModalTrigger, ModalContent, ModalHeader, ModalBody, ModalFooter, ModalClose, Button } from "krds-community-react";

function DeleteConfirm() {
  return (
    <Modal>
      <ModalTrigger render={<Button variant="tertiary">삭제</Button>} />
      <ModalContent>
        <ModalHeader>삭제 확인</ModalHeader>
        <ModalBody>
          선택한 항목을 삭제하시겠습니까?
          이 작업은 되돌릴 수 없습니다.
        </ModalBody>
        <ModalFooter>
          <ModalClose className="krds-btn medium tertiary">취소</ModalClose>
          <Button variant="primary" size="medium">삭제</Button>
        </ModalFooter>
      </ModalContent>
    </Modal>
  );
}
```

**규칙:**
- 행동의 결과를 명확히 설명
- 취소 옵션 항상 제공
- 파괴적 액션은 Primary 버튼으로 (사용자가 의식적으로 클릭)

### 동의

약관 동의를 받는 패턴입니다.

```tsx
import { Accordion, AccordionItem, AccordionTrigger, AccordionPanel, Checkbox, Button } from "krds-community-react";

function ConsentForm() {
  return (
    <div>
      <Checkbox label="전체 동의" />

      <Accordion variant="line">
        <AccordionItem value="terms">
          <AccordionTrigger>
            <Checkbox label="이용약관 동의 (필수)" />
          </AccordionTrigger>
          <AccordionPanel>
            이용약관 전문...
          </AccordionPanel>
        </AccordionItem>
        <AccordionItem value="privacy">
          <AccordionTrigger>
            <Checkbox label="개인정보 수집·이용 동의 (필수)" />
          </AccordionTrigger>
          <AccordionPanel>
            개인정보 처리방침 전문...
          </AccordionPanel>
        </AccordionItem>
        <AccordionItem value="marketing">
          <AccordionTrigger>
            <Checkbox label="마케팅 정보 수신 동의 (선택)" />
          </AccordionTrigger>
          <AccordionPanel>
            마케팅 정보 수신 안내...
          </AccordionPanel>
        </AccordionItem>
      </Accordion>

      <Button variant="primary" fullWidth>동의하고 계속</Button>
    </div>
  );
}
```

### 로그인

```tsx
import { Tab, TabList, TabPanel, Tabs, TextInput, Button, Link, Checkbox } from "krds-community-react";

          <Link href="/find-id">아이디 찾기</Link>
```

## 버튼 계층 규칙

| 계층 | 용도 | 페이지당 개수 |
|------|------|--------------|
| Primary | 가장 중요한 액션 (제출, 확인, 신청) | 1~2개 |
| Secondary | 보조 액션 (취소, 이전) | 필요한 만큼 |
| Tertiary | 저강조 (더보기, 닫기) | 필요한 만큼 |
| Text | 인라인 (다운로드, 공유) | 필요한 만큼 |

## 네비게이션 구조

```
┌─ Masthead (정부 공식 배너)
├─ Header (로고 + 검색 + 메인메뉴)
├─ Breadcrumb (현재 위치)
├─ [SideNavigation] (서브 페이지)
├─ main
│  ├─ [InPageNavigation] (긴 페이지 목차)
│  └─ 본문 콘텐츠
└─ Footer (기관 정보 + 링크)
```
