글자·화면 표시 설정

글자·화면 표시 설정

화면 표시 모드

page title

검색은 사용자가 큰 데이터 집합에서 원하는 정보를 찾을 수 있도록 도와주는 기능이다. 사용자가 무엇을 찾고 싶은지 알고 있는 경우에는 일차적인 정보 탐색 수단으로 사용될 수 있으며, 탐색 수단을 통해 원하는 콘텐츠를 찾지 못하는 상황에는 특정 정보와 관련된 단서를 제공함으로써 사용자가 필요한 콘텐츠를 쉽게 찾을 수 있도록 해준다.

용례

사용하기 적합하지 않은 경우

  • 단일 화면 웹사이트나 서비스 또는 규모가 매우 작은 서비스에서는 통합 검색 없이 정보를 탐색할 수 있다.

유형

통합검색

웹사이트 또는 애플리케이션 전체를 검색하는 것으로 사용자가 서비스에 익숙하지 않은 경우, 검색을 통해 서비스의 데이터 구조와 활용할 수 있는 정보를 더 쉽게 파악할 수 있다. 일반적으로 헤더 영역에 배치되며, 검색 결과는 별도의 결과 화면에서 제공된다.

  • 웹사이트 통합 검색
  • 자료 검색

부분 검색

소규모 웹사이트, 단일 화면 서비스, 특정 정보 목록, 표와 같은 소규모 데이터 집합을 검색하는 데 사용한다. 데이터 집합 상단에 배치되며, 검색 결과는 데이터 집합에 직접 반영된다.

  • 게시물 검색
  • 목록 부분 검색

이용 상황별 플로(Flow)

  1. 1검색어 입력

    • 검색어 입력 도움 받기

      검색어를 입력하는 과정에서 참고할 수 있는 정보를 제공받고 활용하는 과정

    • 검색어 입력과 수정

      검색 키워드의 입력, 수정, 삭제를 수행

  2. 2검색 결과 확인

    • 결과 화면 진입

      검색 결과 화면으로 진입

    • 검색 결과 개괄

      상세 목록, 목록 내 항목 탐색 전 몇 건의 결과가 도출되었는지를 포함하여 전반적인 결과를 파악하는 과정

    • 목록 탐색

      검색 결과 목록의 구성에 대한 파악을 수행

    • 목표 결과 찾기

      목표한 검색 결과에 대한 탐색 수행

  3. 3검색 결과 이용

    • 외부/연계 서비스로의 이동

      링크를 실행하여 연계된 외부 서비스로 이동하거나 서비스 내에서 관련 페이지로 이동하는 과정

    • 상세 내용 확인

      이동 과정 없이 목록 자체에서 필요한 정보를 확인하는 과정

    • 검색 결과 다운로드/저장

      검색 결과를 다른 목적으로 사용하기 위해 파일 형태로 다운로드 받거나 저장

  4. 4재검색

    • 검색 조건 계속 이용

      기존 조건이 적용된 목록을 계속하여 탐색하거나, 조건 중 일부만을 변경하여 원하는 정보를 계속하여 탐색하는 과정

    • 새로운 검색 조건 설정

      기존 조건을 삭제하고 새로운 검색 조건을 설정하거나, 세부 조건 설정 없이 새로운 키워드로 검색을 시도하는 과정

  5. 5상세 검색

    • 상세 검색 기능 찾기

      상세 검색, 정렬/필터링 수단에 대한 탐색 수행

    • 상세 검색 조건 설정

      검색 결과 목록을 원하는 범위로 좁히기 위해 상세 조건을 설정하는 과정

    • 상세 검색 조건의 확인 및 적용

      설정한 상세 검색 조건을 최종적으로 적용하고 적용 결과를 확인

사용성 가이드라인 체크리스트

서비스 패턴의 사용성 가이드라인은 사용자 관점에서의 중요도와 만족도를 기준으로 하여 3개의 적용 수준으로 구분되어 있다. 핵심 서비스에 대한 사용자 경험의 향상을 위해서는 기본적으로 최하위 적용 수준인 필수 수준의 가이드라인을 준수해야 한다.

적용수준(서비스패턴) 가이드라인 확인

적용 수준 필터

검색 기능 찾기
  • 필수 서비스 내 모든 화면에서 통합 검색 기능을 실행할 수 있도록 한다.
  • 권장 통합 검색 기능 실행 버튼은 모든 화면에서 헤더 우측에 일관성 있게 배치한다.
  • 권장 통합 검색 기능은 검색어 입력 필드와 돋보기 버튼의 조합 또는 돋보기 버튼 형태로 사용한다.
  • 권장 부분 검색 기능은 콘텐츠 목록의 수준과 맥락을 고려하여 사용자가 예측할 수 있는 영역에 배치한다.
  • 권장 검색 기능 주변에 불필요한 요소의 배치를 최소화한다.
검색어 입력
  • 우수 사용자가 검색어 입력 행위에 집중할 수 있도록 한다.
  • 필수 스코프 필터를 사용하는 경우, ‘전체’ 옵션을 제공한다.
  • 권장 검색어 입력 필드는 일반적인 검색어가 한눈에 파악될 수 있는 너비로 제공하고 줄바꿈이나 말줄임표를 사용하지 않는다.
  • 필수 통합 검색이 아닌 검색에는 입력 필드에 레이블을 제공한다.
  • 필수 플레이스홀더를 사용하는 경우, 유도 문구 외에 검색에 대한 단서나 레이블을 포함해서는 안 된다.
  • 권장 사용자에게 다양한 방식의 검색어 입력 도움을 제공한다.
  • 우수 검색어 입력 도움은 가능한 사용자가 검색어를 입력하는 시점에 실시간으로 제공한다.
  • 권장 실시간으로 검색어를 제안할 수 없는 경우, 검색 결과 화면에서 보조적인 도움 수단을 제공한다.
검색 결과 확인
  • 우수 검색 결과가 표시되기 전에 로딩 상태에 대한 정보를 제공한다.
  • 필수 검색 결과 화면에 검색어 입력 필드를 유지한다.
  • 권장 검색 결과 화면에 여러 개의 검색어 입력 필드를 사용하지 않는다.
  • 권장 사용자가 재검색을 시도하지 않는 한 검색어 입력 필드에 검색어를 유지한다.
  • 필수 검색 결과 수를 표시한다.
  • 필수 검색어와 관련된 결과가 반환되는지 확인한다.
  • 우수 유사 검색어나 관련된 검색어에 대한 검색 결과를 함께 제공한다.
  • 권장 초기 검색 결과 목록은 정확도 또는 관련도순으로 정렬하여 제공한다.
  • 우수 검색 결과가 지나치게 많이 제공되는 경우 사용자에게 범위를 제한할 수 있는 방법에 대해 안내한다.
  • 필수 검색 결과가 없는 경우, 결과를 명확하게 알리고 적절한 대안을 제공한다.
상세 검색
  • 필수 기본 정렬, 필터 컨트롤을 제공한다.
  • 권장 기본 검색에서 정렬, 필터 컨트롤은 일정한 순서로 제공되어야 한다.
  • 필수 스코프 필터를 사용하는 경우 결과 페이지 상단에 설정된 범위를 명시적으로 표시한다.
  • 우수 정렬, 필터 적용 결과는 가능한 한 사용자가 값을 선택하였을 때 적용한다.
  • 우수 조건이 적용되었음을 확인할 수 있는 명확한 시각적 단서를 제공한다.
검색 결과 이용
  • 필수 사용자가 필요로 하는 주제별로 검색 결과가 제공되도록 설계한다.
  • 권장 검색 주제 분류는 7개를 초과하지 않도록 한다.
  • 권장 주제별 탐색 탭을 사용하는 경우 전체 목록이 가장 첫 번째 탭으로 제공되어야 한다.
  • 우수 전체 결과 목록은 주제별 섹션 구분 없이 결과의 정확도나 관련도 순으로 정렬한다.
  • 권장 제목과 설명은 명확하고 간결하게 제공한다.
  • 권장 항목에 검색 결과 범주 정보가 명확하게 드러나도록 표현한다.
  • 필수 검색어와 일치하는 항목을 강조 표시한다.
  • 권장 기본 검색 결과 목록에 표시되는 항목의 수를 10개 이상으로 제공한다.
재검색
  • 필수 상세 검색 컨트롤을 숨기지 않는다.
  • 필수 사용자가 설정한 상세 검색 조건을 유지한다.
  • 권장 상세 검색 조건의 적용을 일괄적으로 해제할 수 있는 수단을 제공하고 직관적으로 인지할 수 있도록 표현한다.
검색 종료
  • 필수 검색 과업 맥락에서 벗어나 다른 페이지 또는 다른 탐색 수단에 빠르게 접근할 수 있도록 해야 한다.
